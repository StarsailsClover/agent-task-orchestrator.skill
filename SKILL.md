# Step Task Orchestrator

## Overview

修复 Step 原生 Assistant/Stepclaw 的任务拆分缺陷，实现用户请求全生命周期的标准化处理。用于任何需要多模态文件解析、模型动态路由、分级任务拆解与可靠执行、全周期分层记忆管理的复杂任务场景。当用户请求涉及文件/文件夹处理、需要工具调用、任务链路复杂、或需要多步骤协调执行时触发本 Skill。本 Skill 实现用户请求全生命周期的标准化处理流程，解决原生 Assistant 的任务拆分缺陷。核心能力包括：多模态文件解析、模型动态路由、分级任务拆解与可靠执行、全周期分层记忆管理。

## Execution Flow

所有用户请求必须严格遵循以下执行流程：

### Phase 1: 请求入口与前置处理

```
用户请求 → 读取分层记忆库 → 请求标准化 → 文件包含判断 → 模型调度
```

**Step 1: 读取系统分层记忆库**
- 获取历史对话上下文
- 检索同类任务执行案例
- 加载模型配置历史
- 读取用户偏好数据

**Step 2: 请求标准化处理**
- 统一语义表达格式
- 剔除无效冗余信息
- 补全基础语义上下文

**Step 3: 文件包含判断**（分支逻辑）

#### 分支 1: 请求包含文件资源

**3.1 图片文件判断**
- 若为图片文件：
  1. 执行图像理解处理
  2. 完成图片语义提取
  3. 进入文件夹包含判断
- 若为非图片文件：直接进入文件夹包含判断

**3.2 文件夹包含判断**
- 若为文件夹：
  1. 执行文件安全检测
  2. 创建专属工作区（用户未指定则默认）
  3. 拉取文件夹内全部资源
  4. 完成文件语义嵌入
- 若不为文件夹：进入多文件判断

**3.3 多文件判断**
- 若为多文件：
  1. 执行文件安全检测
  2. 创建专属工作区
  3. 完成文件语义嵌入
- 若为单文件：
  1. 执行文件安全检测
  2. 调用工具直接解析文件
  3. **重要**：使用阅读工具时将 Number 浮点数转换为 int32 整数再注入
  4. 完成文件语义嵌入

**3.4 进入模型调度流程**

#### 分支 2: 请求不包含文件资源

1. 执行安全分级检查（内容合规性与风险等级判定）
2. 执行自然语言语义整理（补全语义逻辑、明确核心需求边界）
3. 进入模型调度流程

---

### Phase 2: 模型调度与推理优化

**Step 1: 确认各工具可用性**
- 检测所有可用工具状态
- 验证工具权限配置

**Step 2: 确认 LLM 状态**
- 评估当前模型能力边界
- 确认模型上下文容量

**Step 3: 执行【任务分级判断】**

基于以下维度进行分级：
- 需求复杂度（简单/中等/复杂）
- 是否需要工具/Agent 调用
- 任务链路长度

分级结果映射到 4 个执行分支：

| 难度 | 需要 Agent | 分支 |
|------|-----------|------|
| 简单/中等 | 否 | 分支 1 |
| 简单/中等 | 是 | 分支 2 |
| 复杂 | 否 | 分支 3 |
| 复杂 | 是 | 分支 4 |

---

### Phase 3: 分级任务执行核心流程

#### 分支 1: 简单/中等难度・无需 Agent 调用

```
调用弱/中等 LLM → 直接推理 → 生成回答 → 短期记忆沉淀 → 输出结果
```

**执行步骤：**
1. 调用匹配后的弱/中等能力 LLM 执行直接推理
2. 基于推理结果生成符合用户需求的完整回答
3. 执行本次对话的短期记忆沉淀（请求、推理过程、回答结果）
4. 向用户输出最终结果，流程结束

---

#### 分支 2: 简单/中等难度・需要 Agent 调用

```
启动 HTN 规划器 → 需求清晰度校验 → 构建 DAG → 环境感知 → 执行原子操作 → 实时反思校验 → 操作成功性校验 → 中期记忆沉淀/重规划
```

**执行步骤：**

1. **启动 HTN 分层任务网络规划器**
   - 对用户需求进行原子化任务拆解
   - 明确：任务目标、执行步骤、前置条件、交付标准

2. **执行【需求清晰度校验】**
   - 判断拆解后的任务是否存在：信息缺失、边界模糊、需求不明确
   - **若需求不清晰**：向用户发起定向询问 → 获取补充信息 → 重新执行 HTN 任务拆解
   - **若需求清晰**：进入下一步

3. **构建 DAG 有向无环依赖图**
   - 明确任务执行顺序
   - 定义依赖关系
   - 设定并行执行规则

4. **启动环境感知监听**
   - 实时获取：系统状态、工具可用性、权限配置、资源限制

5. **完成执行准备**
   - 执行环境适配
   - 工具调用准备
   - 执行权限校验

6. **决策最优执行方式**
   - 基于 DAG 依赖图与环境信息
   - 调度对应工具/能力执行原子操作

7. **实时反思校验（执行过程中同步调用中等能力 LLM）**
   - 核对操作结果与预期目标的一致性
   - 检查操作合规性
   - 识别异常风险

8. **执行【操作成功性校验】**
   - **若成功**：
     - 执行中期记忆沉淀（任务拆解方案、执行过程、操作结果、反思内容）
     - 向用户输出最终结果
     - 流程结束
   - **若失败**：
     - 启动 HTN 重规划
     - 基于失败原因、反思结果调整任务拆解方案与执行路径
     - 重新执行，循环直至成功

---

#### 分支 3: 复杂难度・无需 Agent 调用

```
开启 Chain-of-Thought → 异步研讨机制 → 安全检测 → 生成回答 → 长期记忆沉淀 → 输出结果
```

**执行步骤：**

1. **开启 Chain-of-Thought 思维链推理**
   - 完整拆解需求逻辑
   - 梳理推理路径
   - 构建论证过程

2. **启动异步研讨机制**
   - 基于思维链推理结果
   - 多模型交叉校验
   - 补充优化内容
   - 进行风险排查

3. **执行安全检测**
   - 提示注入安全检测
   - 内容合规校验

4. **生成最终回答**
   - 基于校验通过的内容
   - 生成结构化、逻辑完整的回答

5. **执行长期记忆沉淀**
   - 记录：需求、推理全流程、研讨结果、最终回答

6. **向用户输出最终结果**，流程结束

---

#### 分支 4: 复杂难度・需要 Agent 调用

```
强 LLM + HTN 联合拆解 → 需求清晰度校验 → 记忆库检索优化 → 构建全链路 DAG → 异步研讨校验 → 全链路准备 → 分阶段执行 → 全流程实时反思 → 操作成功性校验 → 长期记忆沉淀/全量重规划
```

**执行步骤：**

1. **启动强能力 + HTN 规划器联合拆解模式**
   - 对复杂需求进行分层、分阶段的原子化任务拆解
   - 明确：总目标、阶段里程碑、原子任务、依赖关系、风险预案

2. **执行【需求清晰度校验】**
   - 判断拆解后的全链路任务是否存在：信息缺失、边界模糊、需求不明确
   - **若需求不清晰**：向用户发起定向询问 → 获取补充信息 → 重新执行强 LLM+HTN 联合任务拆解
   - **若需求清晰**：进入下一步

3. **检索分层记忆库**
   - 召回同类复杂任务的历史执行案例
   - 获取成功方案参考
   - 分析踩坑记录
   - 优化当前任务拆解方案

4. **构建全链路 DAG 有向无环依赖图**
   - 明确任务执行顺序
   - 定义依赖关系
   - 设定并行规则
   - 标记里程碑节点

5. **启动异步研讨机制**
   - 对 DAG 执行方案进行：交叉校验、风险评估、路径优化
   - 输出最终可执行方案

6. **完成全链路准备**
   - 执行环境适配
   - 工具链调度准备
   - 全周期权限校验
   - 异常监控配置

7. **分阶段执行**
   - 基于最终执行方案
   - 决策分阶段执行方式
   - 调度对应工具/能力/Agent 执行原子操作

8. **全流程实时反思校验（同步调用强能力 LLM）**
   - 核对操作结果与预期目标的一致性
   - 检查操作合规性
   - 识别风险隐患
   - 实时优化执行细节

9. **执行【操作成功性校验】**
   - **若全链路操作成功**：
     - 执行长期记忆沉淀（任务拆解方案、执行全流程、反思结果、成功案例、优化经验）
     - 向用户输出最终结果
     - 流程结束
   - **若操作失败**：
     - 启动 HTN 重规划
     - 基于：失败原因、强 LLM 反思结果、环境变化
     - 全量调整任务拆解方案与执行路径
     - 重新执行，循环直至成功

## Memory Management

### 分层记忆库结构

| 层级 | 存储内容 | 保留周期 | 检索策略 |
|------|---------|---------|---------|
| 短期记忆 | 当前对话、简单推理结果 | 单会话 | 全量加载 |
| 中期记忆 | HTN 任务方案、执行过程、反思 | 跨会话 | 相似度匹配 |
| 长期记忆 | 复杂任务全流程、成功案例、优化经验 | 持久化 | 语义检索 + 标签过滤 |

### 记忆写入时机

- **短期记忆**：分支 1 执行完成后立即写入
- **中期记忆**：分支 2 操作成功后写入
- **长期记忆**：分支 3、4 执行完成后写入

## Key Implementation Notes

### 文件解析注意事项

**Number 类型转换**：使用阅读工具解析文件时，必须将 Number 浮点数转换为 int32 整数再注入，避免精度问题。

### HTN 规划器核心要素

每个原子任务必须包含：
- 任务目标（Goal）
- 执行步骤（Steps）
- 前置条件（Preconditions）
- 交付标准（Deliverables）

### DAG 依赖图规则

- 必须为有向无环图（无循环依赖）
- 明确并行执行边界
- 标记关键路径节点

### 反思校验检查项

- 操作结果与预期目标一致性
- 操作合规性（权限、安全策略）
- 异常风险识别
- 执行效率评估
